# React

**What it is:** A JavaScript library for building user interfaces out of
**components** — functions that return what the screen should look like for
some data. When the data changes, React re-renders the affected parts of the
page. The whole frontend is React.

## Next things to learn

- What a **component** is — start with `frontend/src/pages/ProductListPage.tsx`.
- **JSX** — the HTML-like syntax inside the JavaScript/TypeScript code.
- **Props** — passing data into a component (like method parameters).
- **State** with the `useState` hook — data that, when changed, makes the
  component re-render.
- **Effects** with the `useEffect` hook — running code (like fetching from the
  API) when a component appears.
- **Single-page app (SPA)** routing with `react-router-dom` — how `App.tsx`
  maps URLs like `/products/3` to page components without full page reloads.
- Rendering lists with `.map()` and why each item needs a `key`.
